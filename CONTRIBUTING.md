# Contributing to ReportMate

Thanks for your interest in improving ReportMate. This file applies across every
repository in the [ReportMate organization](https://github.com/reportmate).

## Licensing of the project

ReportMate is split-licensed:

- **Server** — the API and web dashboard — is **AGPL-3.0**.
- **Clients** (Macintosh and Windows), the **CLI**, the **Terraform modules**,
  and **deployment tooling** are **MIT**.

A separate **commercial license** is available for organizations whose policies
do not permit AGPL. Offering this commercial option is part of how the project
sustains itself, which is why contributions are accepted under the agreement
below.

## Contributor License Agreement

Before a Contribution can be merged, You must agree to the
[Contributor License Agreement](CLA.md). You keep the copyright to Your work and
grant ReportMate the right to license it — including under the project's
open-source licenses and a separate commercial license.

By opening a pull request, You agree to the CLA for that Contribution.

## Sign your commits (DCO)

Every commit must be signed off under the
[Developer Certificate of Origin](https://developercertificate.org/):

```
git commit -s
```

This adds a `Signed-off-by: Your Name <you@example.com>` line certifying You have
the right to submit the work.

## Pull requests

- Branch from the repository's default branch and open a pull request against it.
- Keep each pull request focused on one change; don't mix unrelated work.
- Match the style and conventions of the surrounding code.
- Make sure the project builds and its checks pass before requesting review.
